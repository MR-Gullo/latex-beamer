\vspace{8pt}

\begin{multicols}{2}

%% EMERGING
\begin{tcolorbox}[profbox, colback=emerging!15, colframe=emerging!60,
  title={\colorbox{emerging!80}{\strut\hspace{2pt}EMERGING\hspace{2pt}}}]
\small\itshape Missing algorithms or major gaps in understanding the sorting process
\tcblower
\small
\begin{itemize}
\item Covers fewer than 5 algorithms or skips many steps; hard to follow the sorting process
\item Diagrams are messy or hard to read; missing labels for key parts like sorted/unsorted sections
\item Shows incorrect logic or wrong order of operations for one or more algorithms
\item Submits more than 3 days late without communication, uses wrong file format, or leaves sections incomplete
\end{itemize}
\end{tcolorbox}

\vspace{6pt}

%% DEVELOPING
\begin{tcolorbox}[profbox, colback=developing!15, colframe=developing!60,
  title={\colorbox{developing!80}{\strut\hspace{2pt}DEVELOPING\hspace{2pt}}}]
\small\itshape All algorithms included but some steps unclear or missing detail
\tcblower
\small
\begin{itemize}
\item Covers all 5 algorithms but skips some steps or uses starting lists that hide key operations
\item Diagrams are readable but labels are inconsistent; some symbols not explained
\item Logic is mostly correct but requires effort to follow; some transitions unclear
\item Submits 1--2 days late or requests last-minute extension; PDF organized but some sections rushed
\end{itemize}
\end{tcolorbox}

\columnbreak

%% PROFICIENT
\begin{tcolorbox}[profbox, colback=proficient!15, colframe=proficient!60,
  title={\colorbox{proficient!80}{\strut\hspace{2pt}PROFICIENT\hspace{2pt}}}]
\small\itshape Complete storyboards with clear logic that is easy to follow
\tcblower
\small
\begin{itemize}
\item All 5 algorithms shown from unsorted to sorted with no skipped steps
\item Diagrams are neat with consistent labels for comparisons, swaps, and sorted/unsorted parts
\item Starting lists chosen well to show all algorithm behaviors; logic flows clearly
\item Submits on time as organized PDF, communicates about any delays, all sections complete
\end{itemize}
\end{tcolorbox}

\vspace{6pt}

%% EXTENDING
\begin{tcolorbox}[profbox, colback=extending!15, colframe=extending!60,
  title={\colorbox{extending!80}{\strut\hspace{2pt}EXTENDING\hspace{2pt}}}]
\small\itshape Impressive quality with sub-procedure breakdowns and polished presentation
\tcblower
\small
\begin{itemize}
\item Includes separate storyboards for sub-procedures (find min, partition, shift) referenced in main diagrams
\item Visual design is polished; uses color, spacing, or creative layouts to enhance clarity
\item Demonstrates deep understanding by choosing lists that highlight edge cases or tricky steps
\item Submits quality work early, well-organized sections, clear communication throughout
\end{itemize}
\end{tcolorbox}

\end{multicols}

\vspace{4pt}
\noindent\small\textbf{Assessment Note:} Submit a single organized PDF with one section per algorithm. Each storyboard should sort a list of 7 integers using step-by-step diagrams. Label key features (current terms, sorted/unsorted parts) and processes (compare, swap). Optional: create separate sub-procedure storyboards for complex steps.
